# Moodmold 设计规格（创建宠物灵魂文案优化）

日期：2026-04-27  
主题：创建宠物灵魂模块文案优化（情感向 + 仪式感）

## 1. 背景与目标

当前「创建宠物灵魂」流程具备功能完整性，但在文案叙事上偏技术描述（例如 AI/3D），与用户期望的情感陪伴价值存在语义落差。

本次优化目标：

- 提升「创建宠物灵魂」入口与流程的情感打动力
- 建立统一的“仪式感神圣”语调
- 在不改动页面结构和主流程的前提下完成最小改动

## 2. 设计结论（已确认）

用户已确认以下创意方向：

- 目标优先级：更打动人（情感叙事）
- 情感基调：仪式感神圣
- 文案长度：极简（1 行标题 + 1 行副文案 + 1 个按钮）
- 方案选择：方案 1（庄重但不玄乎）

## 3. 范围定义

### 3.1 In Scope

- 仅优化「创建宠物灵魂」相关文案
- 覆盖触点：
  - 创建入口（Action Sheet）
  - 创建页（`sc-create`）
  - 生成页（`sc-gen`）状态文案
- 保持现有交互流程、布局层级、组件结构不变

### 3.2 Out of Scope

- 信息架构调整
- 新增/删除表单字段
- 新增业务逻辑、后端接口或生成机制
- 动画、插画、图标等视觉素材替换

## 4. 文案策略

### 4.1 语调原则

- 从“技术动作”转向“生命唤醒”
- 使用低认知负担、可共情的高意象词汇
- 避免过度玄学措辞，保持克制与可理解性

### 4.2 关键词体系

- 核心词：`注入`、`唤醒`、`仪式`、`从此`
- 避免高频词：`AI 生成`、`3D`（在用户可见主文案中弱化）

## 5. 逐触点文案替换

## 5.1 创建入口（Action Sheet）

- 原文案：`上传照片，AI 生成专属 3D 数字宠物与情绪档案`
- 新文案：`上传照片与名字，完成一次灵魂唤醒仪式。`

## 5.2 创建页（`sc-create`）

- 标题
  - 原文案：`新建宠物灵魂 ✨`
  - 新文案：`为TA注入灵魂`
- 主按钮
  - 原文案：`生成灵魂 →`
  - 新文案：`开始唤醒仪式`

## 5.3 生成页（`sc-gen`）

- 顶部状态
  - 原文案：`AI 生成中`
  - 新文案：`灵魂唤醒中`
- 进度起始
  - 原文案：`准备中…`
  - 新文案：`仪式准备中…`

## 5.4 生成步骤（5 步）

- 原：`照片已分析` -> 新：`记忆映射完成`
- 原：`性格已建模` -> 新：`心性刻印完成`
- 原：`3D 形象已雕刻` -> 新：`灵体塑形完成`
- 原：`情绪引擎已绑定` -> 新：`情感共鸣建立`
- 原：`实体模型已排队` -> 新：`灵魂载体就绪`

## 6. 一致性检查要点

- 所有用户可见主路径文案应维持同一叙事：创建 -> 唤醒 -> 完成
- 入口与流程内不出现语义冲突（如“仪式”与“AI 生成中”并存）
- 在保留可理解性的前提下，减少生硬技术词

## 7. 验收标准

1. 用户进入创建路径时，可在入口、副标题、按钮感知到统一“唤醒仪式”语调
2. 生成页状态与步骤文案不再出现技术割裂
3. 交互流程与页面结构无变更
4. 文案变更可通过静态替换直接落地，不引入额外逻辑依赖

## 8. 风险与回退

- 风险：部分用户对“仪式感”表达偏好不强，可能更偏“效率导向”
- 对策：文案改动保持最小可回退，必要时可回切到中性版词库

---

本设计文档仅定义「创建宠物灵魂」文案与叙事优化，不包含功能新增与交互重构。
