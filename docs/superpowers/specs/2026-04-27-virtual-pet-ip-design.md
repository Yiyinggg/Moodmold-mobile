# Moodmold 设计规格（Brainstorming）

日期：2026-04-27  
主题：虚拟宠物 IP 化（情感陪伴优先）

## 1. 背景与目标

在不改动现有首页结构的前提下，围绕「把自己的宠物打造成虚拟 IP」进行界面扩展。产品优先级为：

1. 情感陪伴（A）
2. IP 打造（B）
3. 内容变现（C）

本轮范围限定为：

- 保持首页现状
- 新增两个页面：`IP设定`、`内容工坊`
- 底部导航将 `收藏` 替换为 `IP设定`

## 2. 范围定义

### 2.1 In Scope

- 新增页面 `sc-ip`（IP设定）
- 新增页面 `sc-studio`（内容工坊）
- 底部导航入口调整：`收藏` -> `IP设定`
- 在 `IP设定` 页面提供「去内容工坊」主入口
- 内容工坊主能力聚焦「生成宠物动画」

### 2.2 Out of Scope

- 首页信息架构与模块重构
- `商店` Tab 的替换或重命名
- 复杂后端能力（真实发布、交易、订单、支付）
- 动画生成引擎本身（本轮仅做前端交互与状态呈现）

## 3. 信息架构与导航

### 3.1 底部导航调整

- 首页：不变
- 第二个 Tab：由 `收藏` 改为 `IP设定`
- 中间创建按钮：保持现状
- 商店/我的：保持现状

### 3.2 主路径

`首页` -> `IP设定` -> `内容工坊` -> `生成结果操作（保存/发布/复用）` -> `返回首页陪伴`

设计意图：

- 用 `IP设定` 保障角色一致性（先定义再生成）
- 用 `内容工坊` 承接创作与后续变现能力
- 通过「返回首页陪伴」回流到核心价值（情感连接）

## 4. 页面规格

### 4.1 `IP设定`（`sc-ip`）

### 页面目标

把宠物从单次互动对象升级为可持续创作的角色资产。

### 模块结构

1. 顶部区
   - 标题：`IP设定`
   - 副标题：`让TA从宠物变成可持续创作的角色`
2. 角色基础（必填）
   - 名称
   - 物种
   - 年龄感（幼年/成年等）
   - 一句话介绍
3. 性格标签（多选）
   - 示例：`黏人`、`机灵`、`慢热`、`戏精`
   - 展示：当前人格倾向摘要
4. 角色世界观（选填）
   - 口头禅
   - 关系对象（主人/朋友）
   - 常出现的场景
5. 视觉偏好（选填）
   - 主色
   - 配饰
   - 服装风格
6. 底部操作区
   - 主按钮：`保存并去内容工坊`
   - 次按钮：`仅保存设定`

### 关键交互

- 用户修改字段后立即更新本地状态
- 必填未完成时，主按钮仍可点，但先提示缺失项并引导补全
- 保存成功后显示轻提示（toast）

### 4.2 `内容工坊`（`sc-studio`）

### 页面目标

将 IP 设定转化为内容产出，第一优先能力是宠物动画生成。

### 模块结构

1. 顶部区
   - 标题：`内容工坊`
   - 副标题：`把你的IP宠物变成内容资产`
2. 主能力卡（最大视觉权重）
   - `生成宠物动画`
   - 输入项：情绪、场景、时长（15s/30s）
   - 状态：待生成/生成中/生成完成
3. 次级能力卡（占位）
   - 图文内容（即将上线）
   - 周边设计稿（即将上线）
4. 结果操作区
   - `保存`
   - `发布`
   - `复用本次模板`
5. 回流入口
   - `返回首页陪伴`

### 关键交互

- 若 `IP设定` 完整度低，点击主能力时给轻提示：
  - `补全2项设定可提升动画质量`
- 不强拦截，允许继续生成
- 生成完成后展示结果卡与操作按钮

## 5. 数据与状态（前端原型级）

最小状态模型建议：

- `ipProfile`：角色基础信息 + 性格标签 + 世界观 + 视觉偏好
- `ipProfileCompleteness`：设定完整度百分比
- `studioDraft`：本次生成输入（情绪/场景/时长）
- `studioOutputs`：生成结果列表（含状态字段）

本轮可先用内存状态或 `localStorage`，不依赖后端。

## 6. 错误处理与空状态

- `IP设定` 空状态：显示示例文案，降低用户填写成本
- `内容工坊` 首次无内容：展示一键示例模板（如“日常撒娇 15s”）
- 生成失败：展示重试按钮 + 原输入保留
- 网络不可用（若接入接口）：给出离线提示并允许稍后重试

## 7. 验收标准（UI/交互）

1. 底部 `收藏` 成功替换为 `IP设定`，可进入 `sc-ip`
2. `sc-ip` 包含必填+选填模块与双按钮操作
3. `sc-ip` 可以跳转 `sc-studio`
4. `sc-studio` 首屏主入口为 `生成宠物动画`
5. `sc-studio` 存在结果操作区与回流首页按钮
6. 未完善设定时，能看到轻提示但不阻断生成流程
7. 首页结构未被改动

## 8. 实施顺序建议（高层）

1. 先完成导航替换与两个页面骨架
2. 再补齐 `IP设定` 表单与状态
3. 再接 `内容工坊` 的生成流程与结果状态
4. 最后补空状态、失败态和文案打磨
